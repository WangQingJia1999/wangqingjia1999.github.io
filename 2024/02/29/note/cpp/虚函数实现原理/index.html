C++ 虚函数实现原理

发表于 2024-02-29 更新于 2024-03-05 本文字数： 9.6k 阅读时长 ≈ 16 分钟

从汇编角度，分析编译器（本文基于 gcc version 9.4.0）如何实现c++的虚函数机制。

注意：c++标准没有规定必须用虚函数表这种形式实现，只是大多数编译器都会采用这种实现方式。

原理概述：一个类只要声明了一个虚函数，则编译器为该类生成一个虚函数表，虚函数表本质是一个函数指针数组，包含该类所有成员函数的最终入口地址。
每个实例内存一个虚函数表起始地址的虚函数表指针。调用每个成员函数时，查虚函数表，在代码段找到真正的函数入口地址，执行对应汇编代码。

需要明确：虚函数表本身在数据段内，由该类所有实例共享；而虚函数指针在单个对象实例内。

虚函数表指针在哪?

一个指向多态对象的指针，指向成员变量连续内存的起始地址，但由于vptr在成员变量列表最开始位置，因此实际是指向vptr。

多继承虚函数指针怎么分布？

gcc导出vtable

C类继承4个父类，类C本身的虚函数跟在第一个父类指针数组后面，其余父类的虚函数依次排列，每个C类的对象含4个vptr，分别指向C类虚函表数组的四个地址。注意是全指向C类的虚表数组，数组中存的是虚函数地址，不涉及父类虚表。编译期就确定了某个类的虚表分布与构成。
